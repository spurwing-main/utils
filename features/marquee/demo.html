Marquee Feature Demo

This page exercises every part of the marquee runtime. Open your devtools console to watch debug logs emitted by the loader (data-debug="marquee"). Each section below highlights a specific use case or edge case.

01. Basic marquee

A default marquee uses data-marquee and scrolls at 1px per frame. Speed is consistent no matter the container width.

Design tokens Composable APIs Zero legacy fallbacks Idempotent init()

02. Tuned speeds

Speed is declared via data-marquee-speed in pixels per frame (at 60fps).

Slow: 0.5 Perfect for subtle UI surfaces
Fast: 2.75 Smooth even with long copy

03. Interactive content

Focusable children remain interactive in the original DOM. Cloned nodes are hidden from assistive tech and cannot capture focus.

View source Biome powered

04. Dynamic marquees

Add or remove marquees at runtime. The feature exposes Marquee.rescan() to discover new nodes, while attribute tweaks are instant.

05. Scoped rescans

Rescanning a specific subtree only touches marquees in that region. Others remain untouched.

Scoped marquee A Will detach when toggled off
Scoped marquee B Always stays running

06. Responsive sizing

Drag the slider to shrink or expand the marquee container. The runtime skips tiny jitter but remeasures on real width changes and dramatic height shifts.

Resize-aware marquee Only meaningful width deltas Large height shifts refresh too

07. Global API helpers

The loader exposes window.Marquee. Use it directly without imports once the feature initialises.

    Notes

    • The feature honours prefers-reduced-motion — toggle it in your OS to see animations stop.
    • Every marquee is idempotent: repeated attaches simply refresh measurements and speed.
    • No legacy fallbacks or timers are used. `ResizeObserver` handles measurement updates.
